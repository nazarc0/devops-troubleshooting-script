# ops_first_aid.sh

Bash-скрипт первинної DevOps-діагностики Linux-сервера.

## Що робить скрипт

`ops_first_aid.sh` — інструмент швидкого "першого огляду" сервера після
алерту: повільна робота, нестача дискового простору, незрозуміло хто
залогінений, потрібно знайти важкий процес, конфіг-файл або лог, або
безпечно керувати процесом.

Підтримує 6 режимів роботи:

1. **`--users`** — аудит користувачів (список звичайних користувачів,
   активні сесії `who`/`w`; з `--user <імя>` — UID, GID, групи, домашня
   директорія, shell конкретного користувача).
2. **`--report`** — повний системний звіт (користувач, root/sudo, хост,
   дата, uptime, CPU/пам'ять з `/proc`, диски, активні сесії, топ-5
   процесів за CPU/пам'яттю, зомбі/зупинені процеси, підсумковий статус
   OK/WARNING/CRITICAL). **Звіт автоматично зберігається у файл**
   `/var/tmp/ops_reports/report_YYYY-MM-DD_HH-MM-SS.txt`, а також
   оновлюється симлінк `latest_report.txt` на останній звіт.
3. **`--proc`** — моніторинг процесів (топ-10 за навантаженням,
   статистика за станами R/S/D/T/Z; з `--pid <pid>` — власник, PID/PPID,
   статус, команда запуску, ланцюг батьківських процесів через `pstree`).
4. **`--find <директорія> <патерн>`** — пошук файлів за патерном,
   визначення типу через `file`, дерево директорії глибиною 2 через
   `tree` (якщо встановлено).
5. **`--log <файл> [--follow]`** — аналіз логу: кількість рядків,
   останні 20 рядків, опційно живий моніторинг через `tail -f`.
6. **`--pid <pid> --signal <TERM|KILL|STOP|CONT>`** або
   **`--pid <pid> --renice <значення>`** — безпечне керування процесом:
   заборона дій з PID 1, перевірка існування процесу, вимога root для
   TERM/KILL та для підвищення пріоритету (negative nice).

## Вимоги

- Тільки **Linux** (скрипт явно перевіряє `uname -s` і завершується з
  помилкою на інших ОС, оскільки `ps` та `/proc` поводяться інакше на
  BSD/macOS).
- Bash 4+.
- Стандартні утиліти: `ps`, `who`, `w`, `id`, `getent`, `awk`, `grep`,
  `find`, `df`, `wc`, `tail`, `kill`, `renice`, `hostname`, `uptime`.
- Опційно (для розширеного виводу, скрипт коректно працює і без них):
  `tree`, `pstree`, `file`.

## Як запустити

```bash
chmod +x ops_first_aid.sh
./ops_first_aid.sh --help
```

### Приклади команд

```bash
./ops_first_aid.sh --users
./ops_first_aid.sh --users --user test1

./ops_first_aid.sh --report

./ops_first_aid.sh --proc
./ops_first_aid.sh --proc --pid 1234

./ops_first_aid.sh --find /etc "*ssh*"

./ops_first_aid.sh --log /var/log/auth.log
./ops_first_aid.sh --log /var/log/auth.log --follow

./ops_first_aid.sh --pid 1234 --signal TERM
./ops_first_aid.sh --pid 1234 --renice 5
```

## Які команди/концепції з лабораторної використані

`who`, `w`, `cat`/`getent passwd`, `grep`, `awk`, `sed`, `id`, `groups`,
`find`, `file`, `tree`, `df -h`, `ps`, `pstree`, `kill`, `renice`,
`tail`, `wc -l`, читання `/proc/cpuinfo` та `/proc/meminfo`, `uptime`,
`hostname`, `whoami`, перевірка `$EUID` для прав root.

## Технічні рішення

- Шебанг `#!/usr/bin/env bash`.
- Логіка розбита на **функції** (`cmd_users`, `cmd_report`, `cmd_proc`,
  `cmd_find`, `cmd_log`, `cmd_pid_action`, `check_linux`,
  `command_exists`, `is_integer`, `show_help`).
- Підтримка `--help`.
- Валідація аргументів (PID та значення `renice` мають бути числами,
  директорії та файли перевіряються на існування).
- `set -uo pipefail` — без `-e`, оскільки деякі команди (`grep -c`,
  `ps -p` для перевірки) штатно повертають ненульовий код, що не є
  помилкою скрипту.

## Допущення та обмеження

- Скрипт працює тільки на Linux; на macOS/BSD завершується з помилкою.
- Якщо `tree`, `pstree` або `file` не встановлені — скрипт продовжує
  роботу і виводить попередження замість падіння.
- Для відправки сигналів `TERM`/`KILL` та для підвищення пріоритету
  (`renice` з від'ємним значенням) потрібні права root/sudo.
- Звіт (`--report`) зберігається в `/var/tmp/ops_reports/`; якщо цю
  директорію неможливо створити (наприклад, немає прав), скрипт виводить
  попередження і показує звіт лише в терміналі.
- Скрипт не намагається інтерпретувати локалізовані формати дати/часу
  різних дистрибутивів — використовує стандартний `date` та `uptime -p`
  (з фолбеком на звичайний `uptime`, якщо `-p` не підтримується).

## Бонуси, реалізовані в скрипті

- Симлінк `latest_report.txt` на останній згенерований звіт.
- Перевірка наявності `tree`, `pstree`, `file` перед використанням.
- Зрозумілі повідомлення про помилки замість "сирих" падінь bash.
